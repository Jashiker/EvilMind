使用说明

邪恶思潮 v2.0 · 零信任4阶段多Agent谣言核查系统

产品介绍

邪恶思潮是一款面向政企客户的零信任认知战防御系统。系统基于 DeepSeek-v4-pro 国产大模型，采用 4阶段10 Agent协作架构，实现了从指纹提取、多源并行搜索、合议庭裁定的全自动化核查流水线。前端通过 SSE（Server-Sent Events） 实现真正流式推送，每个 Agent 的分析进度实时可见。

系统架构

系统采用严格顺序4阶段管道，阶段间有明确依赖，阶段内最大化并行。所有 Agent 通过统一 LLM 客户端调用 DeepSeek-v4-pro API，每个 Agent 拥有独立的 system prompt 和角色定义。搜索工具（360搜索、Claude CLI、OpenOSINT）通过 chat_with_tools 机制由 LLM 自主调用。

用户输入 → 知识库检索(相似案例匹配)
  → [阶段1] 指纹研判团队: 4专家并行 → 综合研判官 → 恶意评分 + 关键词
  → [阶段2] 三路侦察兵: 3 Agent并行取证 (事实核查 + 深度挖掘 + 来源验证)
  → [阶段3] 合议庭: 3仲裁官并行裁定 → 首席综合判决
  → [阶段4] 品质审核: 报告生成 → 4维质量自评
  → 输出: 综合判定 + 证据溯源链 + 认知处方 + 辟谣卡片

核查流程

1

信息输入

支持两种输入方式：文字粘贴（直接输入待核查文本）和图片审查（上传截图或 Ctrl+V 粘贴，通过百度飞桨OCR提取文字后自动送入核查流水线）。系统首先进行14维指纹特征提取和知识库相似案例匹配。

2

指纹研判团队（阶段1）

4位专家并行分析——指纹对比官提取语言特征，进化树分析师追溯变异历史，意图挖掘官检测认知操纵手法，风险评估官量化舆情风险。综合研判官汇总后输出0-1恶意评分、攻击维度（social_cohesion/data_fabrication/emotion_hijack等）和三组搜索关键词（事实核查关键词、深度搜索关键词、来源验证关键词），分别传递给下一阶段的三路侦察兵。

3

三路侦察兵并行取证（阶段2）

三个Agent同时工作，各自使用专属关键词和工具链。Claude事实核查官侧重逐条事实点精准验证，Claude深度挖掘官侧重事件背景和舆论趋势，OpenOSINT验证官侧重来源可信度评级。每个Agent自主调用搜索工具（360搜索 web_search/Claude CLI claude_web_search_deep/来源验证 verify_source_credibility），通过 chat_with_tools 的多轮工具调用机制完成搜索→抓取→分析。

4

合议庭仲裁（阶段3）

3位仲裁官独立审议全部证据（指纹分析+三路搜索结果），各自给出结论（true/false/partial/unverifiable）和置信度。首席仲裁官拿到3份独立裁定后综合判决：确认多方印证的事实，标记存在分歧的发现，输出最终裁定。

5

报告输出与品质审核（阶段4）

生成包含以下内容的完整报告：综合判定（verdict_false/true/manipulative/suspicious/unknown）、证据溯源网络（vis.js可视化，双击节点打开真实URL）、认知处方（"怎么骗你的" + "怎么防"）、辟谣卡片（一键复制分享）。品质审核官对报告进行4维质量自评（证据充分度、置信合理性、核查完整性、结论严谨性）。

阶段1: 指纹研判团队

管道入口阶段，4位指纹专家并行分析 + 1位综合研判官串行汇总。综合研判官除了输出恶意评分外，还承担关键词提炼职责——从谣言文本中提取三组搜索关键词，精准传递给下一阶段的三个侦察兵。

每个Agent拥有独立工具链，通过 chat_with_tools 多轮工具调用机制自主完成搜索→抓取→分析全流程。指纹官提炼的关键词在此阶段被精准分配给对应Agent。

阶段3: 合议庭仲裁

采用三级裁定 + 首席综合模式，确保判定独立性和一致性：

  1. 3位仲裁官并行独立裁定 — 各自审议全部证据（指纹分析 + 三路搜索发现），独立输出结论和置信度
  2. 首席仲裁官综合判决 — 拿到3份独立裁定，确认多方印证的事实（consensus），标记存在分歧的发现（disputed），输出最终裁定

输出: 总信源数、交叉印证数、证据质量评级(high/medium/low)、综合置信度(%)、共识/争议清单。

裁定结论类型: true（属实）、false（虚假）、partial（部分属实）、unverifiable（无法核实）。最终综合判定: verdict_false / verdict_true / verdict_manipulative / verdict_suspicious / verdict_unknown。

阶段4: 品质审核官

最终阶段包含两个子步骤：

  1. 报告生成 — 汇总全部阶段的输出，生成综合判定、证据链、认知处方、辟谣卡片
  2. 4维质量自评 — 对报告进行自我评估，输出质量分数和维度评分
维度评估内容
evidence_sufficiency证据充分度：搜索结果是否足够支撑结论
confidence_reasonability置信合理性：置信度与证据强度是否匹配
completeness核查完整性：是否遗漏关键核查角度
rigor结论严谨性：判定逻辑是否严密无漏洞

认知处方

每份报告包含面向普通读者的防骗指南：

  • 怎么骗你的 — 拆解谣言使用的具体操纵手法（情绪煽动、数据伪造、权威伪装等）
  • 怎么防 — 针对性的防骗建议和认知免疫力提升

图片审查

支持两种图片输入方式：

  • 上传截图 — 点击"📷 上传截图"按钮选择图片文件
  • Ctrl+V 粘贴 — 在文本框中直接粘贴剪贴板中的截图

图片通过百度飞桨OCR远程API提取文字（支持中英文混合识别），识别出的文字自动填入文本框并送入4阶段核查流水线。前端提供图片预览和清除功能。

OCR认证: API Key + Secret Key OAuth获取access_token（缓存30天）。需在 .env 中配置 BAIDU_OCR_API_KEY 和 BAIDU_OCR_SECRET_KEY。

知识库

三层知识体系：

  • 案例库 — 500+ 条已核查案例，Bigram相似度匹配，命中相似度 ≥0.6 时作为上下文注入各Agent的prompt
  • 模式库 — 8种操纵模式（极端数字、紧急呼吁、情绪触发、声称权威、身份绑定、阴谋暗示、全称否定、健康恐慌等）
  • 指纹特征库 — 14维特征交叉匹配（has_extreme_numbers/has_urgency_call/has_emotional_trigger 等）

知识库不短路管道——即使KB高度匹配（≥0.9），信息仍走完完整的4阶段核查流程。知识库作为辅助上下文（kb_context/kb_hint）注入，不替代实际搜索验证。

可通过 /api/knowledge/stats 查询知识库统计，通过 /api/knowledge/graph 获取知识图谱数据（前端 vis.js 可视化）。

常见问题

Q: 一次完整核查需要多长时间？
A: 约60-120秒。指纹团队5次API调用（4并行+1串行）、侦察兵3次（并行，含多轮工具调用）、合议庭4次（3并行+1串行）、审核2次（串行）。实际耗取决于搜索轮数和网络延迟。

Q: 搜索为什么可能查不到内容？
A: 搜索引擎本身对最新内容的索引有延迟（尤其是刚发生的事件）。360搜索对国内官方源覆盖最优，但对社交媒体（微博/小红书）的内容索引有限。系统通过 Claude Code CLI 搜索作为补充渠道。

Q: 如何确保 LLM 使用了正确的搜索关键词？
A: 指纹研判官提炼的关键词经 _enforce_year_tags() 自动注入年份标签后，写入各侦察兵的 user prompt。v2.0.1 修复了 chat_with_tools 中 user prompt 未加入 messages 的 Bug，确保 LLM 收到明确的搜索指令。

Q: 图片审查报错怎么办？
A: 百度免费API QPS限制2次/秒，等待几秒重试。确认 .env 中 OCR 相关配置正确。系统自动处理 OAuth token 缓存（30天）。

Q: LLM 输出无法解析 JSON？
A: 系统内置双重降级：json_mode → 普通模式手动提取 {...}。同时自动 strip 工具调用标记（<function_calls> 等）。

Q: 知识库匹配到案例后会短路吗？
A: 不会。即使 KB 相似度 ≥0.9，信息仍走完整4阶段管道。知识库作为辅助上下文注入，不替代实际搜索验证。
