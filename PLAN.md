# 真相猎人 — 产品方案

---

## 一、产品定义

### 一句话
**先嗅探恶意意图，再像调查记者一样深挖真相的信息核查Agent。**

### 产品形态
Web应用（FastAPI后端 + 浏览器前端），用户粘贴信息 → 3个Agent接力核查 → 输出报告+辟谣卡片。

### 核心差异
市面上的核查工具都是"来了就查"——不管信息危害大小，平均用力。真相猎人不一样：**先问"这条信息在攻击什么"，邪恶超标的全力深挖，普通的快速判伪。** 像一个老练的调查记者，懂得把时间花在最危险的谣言上。

## 三、3个Agent的职责（用产品语言描述）

### Agent 1：恶意假设官 👁️

**它做什么**：拿到信息先不做核查，而是做一件反直觉的事——假设这条信息的发布者怀有最邪恶的目的，然后看这个目的在攻击什么。

**它回答3个问题**：
1. 如果这条信息是恶意编造的，发布者想达到什么目的？
2. 他在攻击什么？政治稳定、经济信心、还是社会团结？
3. 这条信息危险到什么程度？值不值得启动深度核查？

**输出**：邪恶评分（0-1）+ 攻击维度 + 是否升级

**产品逻辑**：
- evil < 0.4 → 🟢 低危：快速打标，直接出结论
- evil ≥ 0.4 → 🔴 高危：触发Agent 2深度核查

**Demo中的表现**：用户能看到假设官的"思考过程"——"我在假设最邪恶的目的...这条信息在攻击社会对数据的信任...邪恶评分0.75，值得深挖。"

---

### Agent 2：核查记者 🔍

**它做什么**：像一个调查记者，对信息进行溯源、交叉验证、逻辑推演。只在恶意假设官判定"值得深挖"时才启动。

**它的调查步骤**：
1. 提取信息中所有可以验证的事实点
2. 用360搜索找到每个事实点的原始出处
3. 读取原始网页（官方数据、公告原文）
4. 至少2个独立信源交叉验证
5. 逻辑推演：这条信息内在逻辑站得住吗？

**它的职业操守**：
- 找不到原始出处的 → 标注"无法溯源"
- 只有一个信源的 → 标注"来源单一，可信度低"
- 无法验证的 → 明确说"无法验证"，不硬判

**Demo中的表现**：用户能看到记者在搜索、在抓取网页、在比对数据——"360搜索找到3个相关结果...正在读取国家统计局官网...交叉验证完成，2个信源数据一致。"

---

### Agent 3：真相发布官 📋

**它做什么**：把前两个Agent的发现，整理成普通人能读懂的报告。

**它的交付物**：
1. **核查报告**：一句话结论 + 邪恶评分 + 置信度
2. **证据链**：每一步的调查过程和来源，可追溯
3. **认知处方**：它怎么骗你的、你的什么弱点被利用了、以后怎么防
4. **辟谣卡片**：一个可以一键复制/截图分享的精简版辟谣

**Demo中的表现**：用户看到报告逐块生成——先是结论卡片，然后是证据链，最后是认知处方和辟谣卡片。

## 五、技术选型（为什么这么选）

| 技术 | 选型 | 原因 |
|------|------|------|
| 大模型 | DeepSeek-V3 | 国产，中文能力强，支持function calling |
| 搜索引擎 | **360搜索API** | 国产搜索引擎，中文官方数据源索引好，360生态联动 |
| 向量数据库 | ChromaDB | 轻量，支持中文语义检索，无需额外部署 |
| 后端 | FastAPI + SSE | 流式推送，用户实时看到Agent工作过程 |
| 前端 | HTML + Tailwind + Vanilla JS | 零依赖，演示稳定，不出意外 |
| 部署 | 单机 uvicorn | Demo现场一键启动 |

## 六、Demo演示脚本（5分钟）

### 开场（30秒）
> "真相猎人，一个先嗅探恶意、再深挖真相的信息核查Agent。它的核心理念是：不是所有谣言都值得用同样的力气去查。先判断它有多危险，再决定查多深。"

### 案例1：高危谣言深度核查（90秒）
> 输入"中国结婚率不足1%，离婚排长队，年轻人已经躺平了"

展示流式过程：
1. **假设官**：思考中 → "假设发布者试图攻击公众对统计数据的信任...邪恶评分0.75...触发深度核查"
2. **记者**：360搜索 "中国结婚率 国家统计局" → 抓取stats.gov.cn → 实际数据5.4‰ → "不足1%"偏差5.4倍
3. **发布官**：输出核查报告 + 证据链 + 认知处方

> "数据严重失实——5.4‰ vs 声称的不足1%，差了5倍多。但我们不只是告诉你错了——我们还告诉你它是怎么骗你的：极端数字锚定让你感觉'天哪这么低'，实际上连单位都换了。"

### 案例2：低危信息快速判伪（30秒）
> 输入"长期喝纯净水会导致骨质疏松"

1. **假设官**：evil_score 0.25 → 普通健康谣言，不升级
2. 直接出结论 + 防骗建议

> "这条信息没有攻击政治、经济或社会信心的意图，它只是常见的健康伪科学。真相猎人快速判伪，不浪费深度核查资源。这就是分级调查的价值。"

### 案例3：知识库复用（30秒）
> 再次输入结婚率谣言

→ 知识库命中，0.5秒出结果。

> "第二次输入相同信息时，知识库直接命中，秒出结果。这就是记忆机制——核查过的信息不再重复劳动。"

### 走查闭环（60秒）
展示：辟谣卡片 → 一键复制 → 用户反馈按钮

> "完整的闭环：用户得到了可操作的结论、可分享的辟谣卡片，还可以反馈判断是否准确，系统会根据反馈不断修正。"

### 展示360生态（30秒）
> 切换到搜索日志视图，展示"🔍 360搜索(so.com): 找到5条结果"

> "真相猎人的核心搜索引擎是360搜索API。360搜索对国内官方数据源的中文索引覆盖优于Google和Bing——查中国的政策、数据、社会事件，360搜得更准。同时架构预留了与360浏览器、360安全大脑的联动接口。"

### 总结（30秒）
> "真相猎人——先嗅探恶意，再深挖真相。覆盖6类认知操纵手法，3人Agent团队协作，360搜索驱动，完整的核查闭环。谢谢。"
